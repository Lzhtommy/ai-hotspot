# 标签分类 (30 个封闭标签,最多选 5 个)

从以下 30 个标签中为每条新闻选择 1–5 个最相关的。不要创造新标签。标签返回值必须完全匹配(包括中英文、大小写)。

## 主题类 (12 个)
- **模型发布** — 新模型首次公开或开源
- **Agent** — agentic 系统、多步推理、工具调用、自主任务完成
- **编码** — AI 编程助手、代码生成、debug
- **RAG** — 检索增强生成、向量检索、知识库
- **多模态** — 图像、视频、音频理解或生成;跨模态任务
- **安全** — AI safety、alignment、red teaming、jailbreak 防御
- **评测** — benchmarks、evaluation 方法、leaderboards
- **研究论文** — arXiv / 顶会论文发布
- **产品发布** — 应用层产品上线或重大改版
- **融资** — 融资轮次、估值变化
- **收购** — 收购、合并、团队收购 (acqui-hire)
- **政策** — 政府法规、行业标准、许可变化

## 公司/组织类 (11 个)
- **Anthropic** — 涉及 Anthropic 出品、员工、产品
- **OpenAI** — 涉及 OpenAI
- **DeepMind** — 涉及 DeepMind / Google DeepMind
- **Meta** — 涉及 Meta AI / FAIR
- **Google** — 涉及 Google AI (非 DeepMind 部分)
- **微软** — 涉及 Microsoft AI / Copilot / Azure AI
- **国产模型** — 涉及中国公司的模型 (智谱、百川、Minimax、Moonshot、DeepSeek 等)
- **中国AI** — 涉及中国 AI 政策、中国市场、中文能力(但不特指单一模型)
- **伦理** — AI 伦理、社会影响、versioning、bias、fairness
- **基础设施** — GPU、数据中心、云平台、硬件
- **开源** — 权重开源、代码开源、开源许可证

## 技术类 (7 个)
- **推理** — inference 优化、latency、serving
- **训练** — 预训练、后训练、分布式训练技术
- **闭源** — 闭源模型、专有 API
- **微调** — fine-tuning、LoRA、PEFT、instruction tuning
- **工具链** — 开发者工具、框架、SDK (LangChain、LlamaIndex、vLLM 等)
- **开发者** — 面向开发者的内容 (教程、API changelog、最佳实践)
- **应用案例** — 实际落地案例、用户故事、行业应用

## 标签选择原则
1. 精准优先:宁少不滥。一条新闻 1-3 个标签往往比 5 个更准确。
2. 避免重复语义:`模型发布` 和 `产品发布` 不同步出现 (前者是原生模型,后者是应用层)。
3. 公司标签必须出现在新闻主要主体涉及的公司 — 不要因为一次引用就打标签。
4. 优先具体:`国产模型` 比 `中国AI` 更具体,能用前者就不用后者。

## 各标签详细使用规则

### 主题类标签使用规则

**模型发布**
- 适用:首次公开发布的模型(无论是 API 访问还是权重开源)
- 不适用:已有模型的版本 bug fix、已公布模型的价格调整、模型 API 参数变化
- 常见搭配:`Anthropic` / `OpenAI` / `国产模型` + `模型发布`;`开源` + `模型发布`(权重开放时)
- 区别:`产品发布` 指应用层产品(如 ChatGPT 新功能),`模型发布` 指底层模型本身

**Agent**
- 适用:多步推理 agent 框架、工具调用能力发布、agentic workflow 研究论文、自主任务完成系统
- 不适用:单轮对话模型更新、简单 chatbot 改版
- 常见搭配:`编码`(编码 Agent)、`工具链`(Agent 框架)、`研究论文`(Agent 理论)

**编码**
- 适用:专门面向代码的 AI 工具(Cursor、GitHub Copilot、Devin 等)、代码生成 benchmark、代码理解/调试 AI
- 不适用:通用模型刚好在 HumanEval 上有不错成绩(用`评测`代替)
- 常见搭配:`Agent`(编码 Agent)、`产品发布`(编码工具上线)

**RAG**
- 适用:检索增强生成新方法、向量数据库更新、知识库构建框架、embedding 新模型
- 注意:embedding 模型更新如 Voyage AI 或 OpenAI 的新 embedding 模型应打 `RAG` 标签

**多模态**
- 适用:图像/视频/音频输入或输出的模型或工具
- 不适用:纯文本模型即使在某些 benchmark 上对比了多模态模型
- 常见搭配:`模型发布`(多模态模型)、`研究论文`(多模态研究)

**安全**
- 适用:AI safety 研究、alignment 方法、jailbreak 攻击/防御、有害内容过滤、模型审计
- 不适用:一般性的"AI 可能有风险"评论文章
- 常见搭配:`研究论文`(安全研究)、`政策`(安全法规)、`评测`(安全 benchmark)

**评测**
- 适用:新 benchmark 发布、现有 benchmark 刷新、leaderboard 更新、评测方法论文
- 注意:某模型在 benchmark 上的成绩新闻,如果主角是模型本身,用`模型发布`而非`评测`

**研究论文**
- 适用:arXiv 论文发布、顶级会议论文(NeurIPS/ICML/ICLR/ACL/CVPR 等)
- 不适用:博客文章、公司技术报告(除非同时发布 arXiv 论文)
- 技巧:有代码 + 有 arXiv = 同时打`研究论文`和相应主题标签

**产品发布**
- 适用:面向用户的产品上线(ChatGPT 新功能、Notion AI、Adobe Firefly 等)
- 不适用:底层模型发布(用`模型发布`)、API changelog(用`开发者`)
- 区别:`模型发布`是 API/权重层面,`产品发布`是用户界面/体验层面

**融资**
- 适用:风险投资融资、战略投资、上市(IPO)、SPAC 合并
- 不适用:收购(用`收购`)、政府补贴(用`政策`)

**收购**
- 适用:A 公司收购 B 公司、团队收购 (acqui-hire)、合并
- 注意:`收购`和`融资`互斥,不同时使用

**政策**
- 适用:政府法规(EU AI Act 等)、行业自律标准、许可证变化、监管机构调查
- 常见搭配:`安全`(安全相关法规)、`伦理`(伦理相关法规)、`中国AI`(中国 AI 政策)

### 公司/组织类标签使用规则

公司标签遵循"主要涉及"原则:
- 新闻的主要主体是该公司 → 打标签
- 文中提到该公司作为比较对象 → 不打标签
- 示例:"Anthropic 发布 Claude 3.5" → 打 `Anthropic`; "OpenAI 发布 GPT-5,超越 Claude 3.5" → 只打 `OpenAI`

**国产模型 vs 中国AI 区分**:
- `国产模型`:新闻主角是中国公司开发的具体 AI 模型(DeepSeek-V3、Qwen 2.5、智谱 GLM-4 等)
- `中国AI`:新闻关于中国 AI 生态、政策、市场趋势,或中文语言能力的通用讨论

### 技术类标签使用规则

**推理 vs 训练**:
- `推理`:inference 阶段的优化(vLLM 性能提升、量化方法、KV cache 优化、serving 框架)
- `训练`:训练阶段(预训练数据、后训练方法、RLHF、DPO、分布式训练技术、计算优化)

**闭源**:
- 适用:明确不开源权重的模型或服务
- 注意:`闭源`和`开源`互斥,根据新闻内容选其一或都不选

**微调**:
- 适用:LoRA、QLoRA、PEFT、instruction tuning、domain adaptation、few-shot fine-tuning
- 不适用:通用模型发布即使包含了大量 RLHF 训练(用`训练`代替)

**工具链 vs 开发者**:
- `工具链`:具体的框架、库、SDK(LangChain、vLLM、HuggingFace Transformers、Ollama 等)
- `开发者`:面向开发者的教程、API 使用指南、最佳实践、SDK changelog

## 标签组合模式 (常见场景)

以下是高频出现的标签组合,供参考:

| 场景 | 典型标签组合 |
|------|------------|
| 主要实验室旗舰模型 | `模型发布` + `[公司标签]` + `闭源` |
| 大型开源模型 | `模型发布` + `开源` + `[公司标签]` |
| 中国开源模型 | `模型发布` + `国产模型` + `开源` |
| 编码 Agent | `Agent` + `编码` + `产品发布` |
| 推理框架更新 | `工具链` + `推理` + `开发者` |
| 安全研究论文 | `安全` + `研究论文` |
| 政府 AI 法规 | `政策` + `安全` 或 `政策` + `伦理` |
| 融资新闻 | `融资` + `[若涉及中国公司则 中国AI]` |
| Embedding 模型 | `模型发布` + `RAG` |
| 多模态研究 | `多模态` + `研究论文` 或 `多模态` + `模型发布` |
