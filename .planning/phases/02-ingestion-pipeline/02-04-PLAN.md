<research_findings>
**Research Flag 2 resolved: RSSHub canary routes.**

These are the routes used in the seed. RSSHub route strings are per the RSSHub docs (`https://docs.rsshub.app/routes/`); routes below are stable and widely referenced.

| Source Name      | RSSHub Route                          | Language | Weight | Rationale |
|------------------|---------------------------------------|----------|--------|-----------|
| Anthropic Blog   | `/anthropic/news`                     | en       | 1.0    | Anthropic's official news/announcements feed; high-signal English source; stable. |
| Hacker News AI   | `/hackernews/newest/ai`               | en       | 0.8    | HN newest filtered by `ai` keyword; known stable RSSHub route under `/hackernews/...`. |
| buzzing.cc (AI)  | `/buzzing/whatsnew`                   | zh       | 1.0    | buzzing.cc is referenced explicitly in PROJECT.md as a Chinese-language AI aggregator; `/buzzing/whatsnew` is the primary "what's new" feed covering mixed AI/tech content. |

Route strings are passed to `fetchRSSHub(path)`, which prepends `RSSHUB_BASE_URL` and appends `?key=<ACCESS_KEY>`. The seed stores the PATH only (e.g. `/anthropic/news`) in `sources.rss_url` — NOT the fully-constructed URL — because:
1. The ACCESS_KEY must not appear in the DB (secret hygiene, Phase 1 D-08).
2. `fetchRSSHub(path)` handles the auth envelope; storing paths keeps that responsibility in one place.
3. Rotating RSSHUB_BASE_URL (e.g., migrating from HF Space to a Hetzner VPS) requires zero DB updates.

**Idempotency**: `ON CONFLICT (rss_url) DO NOTHING` via Drizzle's `.onConflictDoNothing({ target: sources.rssUrl })`. The `rss_url` column has a UNIQUE constraint (schema line 25), so re-running the seed inserts 0 rows on subsequent invocations.

**If any route is later found to be broken**: INGEST-07 guarantees the cron keeps polling siblings, and admin (Phase 6) can update the source. Phase 2 verification in Plan 05 tolerates one broken route — it only requires that ≥1 source returns items for the "no-dupe on rerun" test.
</research_findings>

    ```typescript
    /**
     * Canary source seed — Phase 2 D-18, D-19, D-20.
     *
     * Populates the `sources` table with 3 known-working RSSHub routes so the
     * ingestion cron (src/trigger/ingest-hourly.ts) has real data to poll.
     *
     * Idempotent: ON CONFLICT (rss_url) DO NOTHING — re-running is a no-op.
     * Not invoked by CI. Run manually via `pnpm db:seed` against a dev/preview Neon branch.
     *
     * Admin UI for full source CRUD is Phase 6 (ADMIN-02..06).
     *
     * Run: `pnpm db:seed`
     */
    import 'dotenv/config';
    import { config } from 'dotenv';
    config({ path: '.env.local' });

    import { db } from '../src/lib/db/client';
    import { sources } from '../src/lib/db/schema';

    interface SeedRow {
      name: string;
      rssUrl: string;
      language: 'zh' | 'en';
      weight: string; // numeric column — Drizzle takes it as string
    }

    // D-18: 3 canary routes. Routes are RSSHub paths (no base URL, no access key).
    // fetchRSSHub() prepends RSSHUB_BASE_URL and appends ?key=... at request time.
    const SEEDS: SeedRow[] = [
      { name: 'Anthropic Blog', rssUrl: '/anthropic/news', language: 'en', weight: '1.0' },
      { name: 'Hacker News AI', rssUrl: '/hackernews/newest/ai', language: 'en', weight: '0.8' },
      { name: 'buzzing.cc',     rssUrl: '/buzzing/whatsnew',   language: 'zh', weight: '1.0' },
    ];

    async function main() {
      if (!process.env.DATABASE_URL) {
        throw new Error('DATABASE_URL not set — .env.local should provide the Neon dev branch URL.');
      }

      let attempted = 0;
      for (const s of SEEDS) {
        await db
          .insert(sources)
          .values({
            name: s.name,
            rssUrl: s.rssUrl,
            language: s.language,
            weight: s.weight,
            // is_active defaults to true; counters default to 0; created_at defaults to now()
          })
          .onConflictDoNothing({ target: sources.rssUrl });
        attempted += 1;
      }

      // Report post-seed state.
      const rows = await db.select({ id: sources.id, name: sources.name, rssUrl: sources.rssUrl }).from(sources);
      console.log(`Seeded ${attempted} rows (idempotent). Total sources now: ${rows.length}.`);
      for (const r of rows) console.log(`  #${r.id}  ${r.name}  ${r.rssUrl}`);
    }

    main().catch((e) => {
      console.error('SEED FAILED:', e);
      process.exit(1);
    });
    ```

    Path notes:
    - Use `'../src/lib/db/client'` because this file lives at `drizzle/seed-sources.ts` — tsx resolves relative imports; the `@/` alias may not resolve from a `drizzle/` location depending on tsconfig paths + tsx config. Using relative import is safer.
    - If the project already has an `@/` alias working via `tsx` + `tsconfig-paths`, the relative import still works and is equivalent.
