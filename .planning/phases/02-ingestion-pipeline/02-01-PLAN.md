<objective>
Add a nullable `published_at_source_tz TEXT` column to the existing `items` table — the only schema change Phase 2 needs — and push it to the live Neon dev branch so all downstream ingestion work (Plan 03 `fetch-source` task) can write real data to it.

Purpose: D-11 requires preserving the RSS entry's original timezone offset as a string alongside the UTC `published_at` instant. Without the live column in the database, inserts from `fetch-source` would fail at runtime with `column "published_at_source_tz" does not exist` even though typecheck + build would both pass (TypeScript types come from schema.ts, not the live DB).

Output: Modified `src/lib/db/schema.ts`, generated `drizzle/0002_add_published_at_source_tz.sql` (+ journal + snapshot files), and a live Neon dev branch containing the new column.
</objective>
