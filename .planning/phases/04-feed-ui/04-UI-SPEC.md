# Phase 4 — UI Design Contract

> Visual and interaction contract for the anonymous Feed UI (精选 / 全部 AI 动态 / 收藏-placeholder / item detail + WeChat OG). Generated by gsd-ui-researcher from vendored Claude Design bundle at `.design/feed-ui/`. Verified downstream by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual port of vendored Claude Design bundle; Tailwind v4 `@theme` as token surface) |
| Preset | not applicable — tokens copied verbatim from `.design/feed-ui/project/ds/colors_and_type.css` into `src/app/globals.css` (CONTEXT D-04) |

## Spacing Scale

Declared values (4pt base — matches `.design/feed-ui/project/ds/colors_and_type.css` lines 98–110). All values are multiples of 4.

| Token | Value | CSS var | Usage |
|-------|-------|---------|-------|
| xs | 4px | `--space-1` | Icon gaps inside chips, dot sizing, `--radius-xs` alignment |
| sm | 8px | `--space-2` | Tag chip gaps, card action-bar icon gaps, badge horizontal padding |
| md | 12px | `--space-3` | Card internal row gaps, pipeline-card padding, cluster-sibling gap |
| md+ | 16px | `--space-4` | Default card-content vertical rhythm (summary → amber callout), sidebar section padding |
| lg | 20px | `--space-5` | Card padding-y (comfortable density, alongside 22px padding-x per FeedCard `p.card = '18px 22px'`) |
| xl | 24px | `--space-6` | Timeline group margin-bottom (`Timeline` uses 28px — exception below), main column vertical padding |
| 2xl | 32px | `--space-8` | Main column horizontal padding (`Timeline` `'24px 32px 80px'`), FeedTopBar `padding: 18px 32px 0` |
| 3xl | 40px | `--space-10` | EmptyState top/bottom gutter (scaled from design's 96px for in-card empties) |
| 4xl | 56px | `--space-14` | Sidebar+main gutter anchor (reserved; not used in Phase 4 proper) |
| 5xl | 72px | `--space-18` | Timeline bottom padding slot (below last group) |
| 6xl | 96px | `--space-24` | `EmptyState` outer padding (design lines 377–401: `'96px 32px'`) |

**Exceptions (documented & allowed):**

1. **Sidebar width = 224px** (CONTEXT D-08, `sidebar.jsx` line 102). Not on the 4pt scale intentionally — matches the vendored design byte-for-byte; ports to Tailwind v4 as `w-[224px]`.
2. **Card padding = `18px 22px` at comfortable density** (`feed_card.jsx` line 150). Vertical 18px is **not** on the 4pt scale; preserved to match the visual contract exactly. Encoded as `py-[18px] px-[22px]` or as a `--card-padding-*` token in `@theme`.
3. **Top-meta-row card gap = 10px**; **cluster-trigger padding `6px 10px`**; **summary margin = 10px**; **card actions border-top padding-top = 12px**. These sub-grid values trace back to the design; ported as-is. Treat them as internal micro-spacing (not exposed as spacing tokens — they are card-layout constants in `feed-card.tsx`).
4. **Tag chip height = 20px**; **button heights 28 / 34 / 40px** (sm / md / lg per `primitives.jsx` `Button`). 28px is off-scale; preserved.
5. **Hotness-bar height = 3px**; **pipeline-progress-bar height = 3px**; **sidebar NavRow height = 30px**; **sidebar search-input height = 30px**. Off-scale, preserved.
6. **Timeline group margin-bottom = 28px** (`feed_views.jsx` line 175). Off-scale but preserved — rationale is optical grouping between hour blocks.

These exceptions are carried because the vendored design is pixel-authoritative (CONTEXT D-01 "Recreate pixel-perfectly"). The Tailwind v4 `@theme` block stays on the 4pt scale; the off-scale values live as literals inside component files with a code comment citing the design source.

**Minimum touch target:** 40px for IconButton used on mobile (≥375px viewport); desktop retains the design's 30px. Implementation: `IconButton` accepts a `size` prop; the card action-bar renders `size={40}` on `max-lg:` (Tailwind `md:size-[30px]` flip). FEED-07 (responsive ≥375px) — planner's discretion on tactic; this is the baseline assumption.

### Cluster-trigger monogram palette (CONTEXT's "Claude's Discretion" resolved)

The design hardcodes three swatches: `#D4911C / #2558B5 / #E4572E`. **Decision:** keep the three design swatches verbatim when cluster-member count ≥ 3; when member count < 3, show only the first N squares (amber, then blue). Do **not** use per-cluster source colors — the design's 3-swatch constant is a visual motif, not a data visualization, and swapping per-cluster colors would make the motif noisy. `#E4572E` is an intentional "off-palette terracotta" that exists only inside this decorative indicator — it is **not** added to the token set; it lives as a literal value in `cluster-trigger.tsx`.

### Destructive actions

**None in Phase 4.** The only irreversible action that would appear is user-account deletion — Phase 5+ concern. The dislike IconButton (`x` icon, `tone="danger"`) is **not** destructive semantically — it is a preference signal that Phase 5's `votes` table records. No confirmation dialog needed; the red coloring is a visual cue, not a destructive-intent warning.

### Placeholder-button feedback (visual-only buttons)

The `导出` and `手动同步` buttons render visibly but are **disabled** (Button `disabled` prop set) with `title="Phase 6 开放"`. They are not hidden — hiding would hide the information architecture from users; showing-but-disabled is honest.

## Accessibility Contracts

Not a separate template section but load-bearing for downstream consumers; the checker enforces them.

- **Focus ring:** `:focus-visible` uses `--shadow-focus` (`0 0 0 2px var(--paper), 0 0 0 4px var(--accent-500)`) — already declared in `colors_and_type.css` line 88. Applies to all interactive elements including cards (when the whole card becomes a link at Phase 4.x — see below). Never suppress focus styles.
- **Card→detail navigation:** FeedCard title is wrapped in `<Link href="/items/[id]">` so the whole card is *not* a link (avoids nested-interactive-elements since the card contains IconButtons + ClusterTrigger + external-link). Planner implements this as a title-linked pattern, with the external-link IconButton opening the source URL in a new tab (per `feed_card.jsx` line 347).
- **Keyboard:**
  - ClusterTrigger is a `<button>` (matches design) — activates on Enter / Space.
  - ClusterSiblings list items are `<a>` links to their respective `sources.url` (external, new tab with `rel="noopener noreferrer"`).
  - Login prompt modal traps focus and closes on `Escape`. First focusable element receives focus on open.
  - Filter popover is keyboard-reachable via the `过滤` button; tag chips inside are `<button>`s with `aria-pressed`.
- **Screen readers:**
  - Icons inside IconButton must have a `title` attribute (already in design) AND an `aria-label` mirroring the title: `aria-label="收藏"`, `aria-label="Like"`, `aria-label="Dislike"`, `aria-label="打开原文"`.
  - Decorative icons (source-monogram initial, HOT chip, dividers) have `aria-hidden="true"`.
  - SourceDot is decorative — the source name text sits adjacent in DOM and provides the semantic label.
  - Score badge: `<span role="meta" aria-label="热度评分 {score}/100">`.
  - Cluster trigger: `aria-expanded={expanded}`, `aria-controls="cluster-siblings-{id}"`; ClusterSiblings wrapper gets matching `id`.
  - `官方` mini-chip renders as text (not an icon), already accessible.
- **`prefers-reduced-motion`:** honored via a global `globals.css` block: `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { transition-duration: 0ms !important; animation-duration: 0ms !important; } }`. The only Phase 4 motion is the 120ms hover transition on card borders + buttons + nav rows — all reduce to instant under this media query.
- **Color contrast:** spot-checked against WCAG AA:
  - `--ink-900` (#0B0B0C) on `--paper` (#FAF8F4): ratio ≈ 18.5:1 — AAA.
  - `--ink-700` (#3A3833) on `--surface-0` (#FFFFFF): ratio ≈ 11.1:1 — AAA.
  - `--fg-3` (#5C584F) on `--paper`: ratio ≈ 7.8:1 — AAA.
  - `--accent-700` (#8F5D0A) on `--accent-50` (#FCF3E0): ratio ≈ 6.1:1 — AA passes at small text, AAA at large.
  - `--fg-4` (#807A6D) on `--paper`: ratio ≈ 4.2:1 — AA at large text only; **restricted to non-critical captions** (domain footers, middots, disabled nav badges) where this restriction holds.
- **Mobile touch targets:** The card action-bar IconButton size bumps from 30×30 (design default) to 40×40 below the `lg` breakpoint (1024px) per the spacing-exceptions row. Tap-target minimum per WCAG 2.5.5 is 44×44; at 40×40 we are one point under — the cards' action-row includes sufficient padding between buttons (4–8px) so the physical target area is 44×44 including surrounding void.

## Responsive Contract

FEED-07: "Layout renders correctly on a 375px-wide mobile viewport and on desktop."

**Breakpoints:**

| Token | Width | Behavior |
|-------|-------|----------|
| base (mobile) | 375–767px | Sidebar collapses; top-bar shows a `menu` icon button on the left that opens sidebar as a **full-height left drawer** (fixed, width 224px, slides in from left over a 40% paper-opacity backdrop). Main column padding reduces to `16px 16px 64px`. Timeline `max-width` is viewport width. Card padding drops to compact (`14px 16px`) at `<sm`. FeedTopBar buttons (`过滤`, `导出`, `手动同步`) collapse to icon-only (no label text) at `<sm`. |
| sm | 640–767px | Same as base; cards return to comfortable padding. |
| md | 768–1023px | Same as sm; tablet layout — sidebar still drawer-behind-menu (no fixed sidebar on tablets; the 224px sidebar is a desktop affordance). Main column max-width increases to 720px. |
| lg | 1024px+ | **Desktop layout — sidebar fixed at 224px** (matches design 1:1). Main column uses design's `24px 32px 80px` padding + `max-width: 920px` center-aligned (`feed_views.jsx` line 171). |
| xl | 1280px+ | No layout change; wider viewports keep 920px content column centered with expanded gutters. |

**Planner's resolution of CONTEXT D-28's "mobile collapse tactic" discretion item:** left-drawer pattern (above) over `lg:grid-cols-[224px_1fr]` collapse-to-topbar-menu pattern. Rationale: the sidebar carries navigation + pipeline-status + user-chip which are material; squeezing them into a topbar on mobile loses pipeline-status visibility. A drawer preserves the IA verbatim. Implementation: `Sidebar` component renders `position: fixed; inset-y: 0; left: 0` at mobile sizes with `translate-x-[-100%]` closed / `translate-x-0` open; state in a `Client` wrapper holding `isOpen` state; closes on route change.

**Critical non-collapse:** card action-bar never stacks — it's always horizontal. Below 375px (out of scope), items overflow is acceptable.
