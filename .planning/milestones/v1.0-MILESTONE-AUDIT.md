### ⚠ Warnings (non-blocking)

**W1. FEED-10 operational gap (env provisioning, not code):**
`cache-invalidate.ts:52-59` silently no-ops the ISR flush leg if `REVALIDATE_SECRET` or `NEXT_PUBLIC_SITE_URL` is missing. These vars must be set in **both** Vercel env **and** Trigger.dev env vault — `.env.example:88` documents this, but `trigger.config.ts` does not enforce it. Redis flush always runs; if env vars missing in Trigger.dev vault, CDN serves stale HTML up to 5 minutes past cluster refresh until ISR's own revalidate=300 kicks in. **Acceptance criterion for closure:** add to docs/observability.md or operational runbook.

**W2. Drizzle migration snapshot gap:**
`drizzle/meta/` contains `_journal.json` + snapshots 0000-0003 only. Snapshots 0004 (auth) and 0005 (admin ops) are absent, yet the journal lists all 6 migrations. Runtime behavior unaffected (SQL applied successfully per Phase 5/6 live verify). Impact: `drizzle-kit generate` for the next migration (0006) will diff from 0003's schema, producing incorrect output. **Remediation:** regenerate snapshots for 0004 and 0005 from current schema.ts state.
